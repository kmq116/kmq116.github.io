Vue 项目编译 .vue 文件

对项目进行推断，找出编译的包

首先，Vue 项目的创建过程是使用 Vue CLI 创建的，我们进入到 cli 源码中

https://github.com/vuejs/vue-cli 点击进入 github，然后按下键盘的 . 按键，可以打开 github.dev 在线编辑器。

在项目根目录下打开 vue-cli/packages/@vue/cli 文件夹，此文件夹就是存放 vue-cli 源码的。

根据写脚手架的经验，可以顺利的找到 @vue/cli/bin/vue.js 文件

我们的目的是找到为什么 vue 项目可以编译 .vue 文件，以及是怎么编译的，这个时候不妨思考一下，我们使用 vue create <app-name> 命令创建的时候，项目是自动生成好的，生成之后我们也没去增加额外的配置，但是对于其中的 .vue 文件是可以直接启动本地服务跑起来的，那么其中的玄机就在 create 命令中。

顺着代码去找，可以找到 @vue/cli/bin/vue.js 文件下的 create 命令引用了 cli 文件夹下的 @vue/cli/lib/create.js ，而 create.js 文件中的文件引用了同文件层级下的 Creator.js

这样一段代码一定会被执行
