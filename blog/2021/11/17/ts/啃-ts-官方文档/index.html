语法

JS 语法是合法的 TS (参考 ☞ 指)refers

根据其语法，TS 不会认为 JS 语法有任何错误，这意味着你可以在 TS 文件中写 JS 文件，完全不需要担心有错误。

类型

然而，TS 是类型的超集，意味着他对一些值能不能使用添加了一些规则，早先提到的 obj.length 并不是一个语法错误，而是一个类型的使用方式错误

你可能真的 打算intend to 用数字除以一个数组，也许 perhaps 只是想看看会发生什么，但是大多数情况下，这都是一个编程错误，TS 的 意愿 designed 是在程序书写正确的同时，仍然捕获更多可能的常见错误，稍后会了解到你可以在代码中怎么配置 TS 的检查严格程度

如果你将一些代码从 JS 文件中移动到一个 TS 文件中，你可能看到很多类型错误，这取决于代码如何写， 这些可能是 合理的legitimate 问题，TS 过于overly 保守conservative 了

通过本指南，我们将会 展示 demonstrate 如何添加各种 (various 多方面的；多样的) 的语法 消除 eliminate 这些错误

运行时行为

TS 也是一个运行时语言，保留 preserves 了 JS 的运行时行为， 例如 除以 0 会是 Infinity 而不是一个运行时报错，一个 原则 principle TS 永远不会改变 JS 代码的运行时行为。

这意味着，如果你从 JS 代码移动到 TS 里，是能保证 guarantee 运行的方式是相同的，即使 TS 认为有类型错误

保持相同的运行时行为是 TS 的基本承诺，这就意味着你可以在两种语言之间 迁移 transition(转换)，而不用担心有些 细微subtle 的不同会让你程序停止工作。

清除类型 Erased Types

粗略 Roughly 的讲，一旦 once TS 编译器完成对你代码的检查，他就会在编译完成的代码里清除类型，这意味着一旦你的代码完成编译，结果就是没有没醒信息的纯 JS 代码。

这也意味着虽然根据类型 推断inferred ，但 TS 不会改变你程序的行为
本质上 bottom line,当你在编译时看到类型错误，类型系统本身并不会影响到你程序是怎么工作的

最后，TS 没有提供运行时的其他库，你的代码将会和用 JS 的标准库一样，因此，没有需要学习的 TS 框架细节

学习 JS 和 TS

我们常常看到一个问题，学 TS 还是 JS?

这个问题的答案是，你不需要学习 JS 就可以学习 TS, TS 和 JS 共享语法和运行时行为，所以你了解任何 JS 的内容都能帮助你同时学习 TS

有许多许多的资源帮程序员学习 JS ,如果你写 TS 程序就不应该忽略他们，例如，Stack Overflow 上有关 JS 的问题是 TS 问题的 20 倍， 但是所有的 JS 问题都可以应用在 TS 上。

如果你自己搜索如何在 TS 中对数组进行排序，记住：TS 是带有类型检查的 JS 运行时，你在 TS 中对数组进行排序的方式和你在 JS 中的排序方式是一样的，如果你 立刻 directly 就找到了 有关 TS 的资源，那很棒。但不要限制自己以考虑您需要关于如何完成 accomplish 运行时任务的日常问题的特定类型的答案。
