为 JS 程序员提供的 TS

TS 和 JS 有些不寻常的 关系 relationship ，TS 提供了 JS 的所有功能，和其中的附加层：TS 的类型系统

例如，JS 提供了 语言 原始的 primitives 像 number``string 这种东西，但是它没有一直检查这些，而 TS 做了这些检查

(consistently 一贯的 assigned 分配)

这意味着已经存在的 JS 正在工作的代码也是 TS 代码，最主要的好处是 TS 可以对你代码中预期之外的行为进行高亮显示，降低了 bug 的出现率。

这教程提供了 TS 简短的概述，重点放在 TS 的类型系统上。

类型推断 (types by inference)

TS 知道 JS 语言，并在多数情况下为你生成类型，例如创建一个变量并给他分配一个 特殊的 particular 值，TS 将会使用这个值的类型作为变量的类型。

通过理解 JS 如何工作，TS 能接受 JS 代码，但是会构建类型系统。TS 不需要在你代码中 显式 explicit 的添加额外字符，就能知道上面 Hello World 是一个 string类型。

你可能在 vscode 中写 JS,并且需要编辑器自动完成，vscode 底层使用的 TS 代码使得它与 JS 工作更容易。

结构类型系统 Structural Type System

TS 核心规则之一是聚焦于类型的形状的类型检查，有时候这被称作 “duck typing” or “structural typing”

在结构类型系统里，如果两个对象有相同的形状，他们会被认为是统一个类型

point 变量从未被声明为 Point 类型, 然而，TS 在 Point 的类型检查上 通过对比 point 的形态。他们具有相同的形态，所以代码检查通过。

形态匹配值需要对象字段的子集来进行匹配。
